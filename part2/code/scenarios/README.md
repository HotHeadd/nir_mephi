# Модельные сценарии валидации (глава 3)

Пять сценариев из подраздела 1.6 методики. Каждый — отдельная модельная цель
или режим запуска target. Назначение: проверить корректность отлова,
отсутствие ложных срабатываний и базовую атрибуцию.

1. legit_write    — легитимная запись ДО фиксации области (ложных срабатываний быть не должно).
2. foreign_module — запись из посторонней библиотеки (атрибуция до модуля).
3. foreign_func   — запись из другой функции того же модуля (атрибуция до функции).
4. buffer_overflow— затирание critical_flag переполнением соседнего буфера.
5. control_unguarded — обращение к НЕохраняемой области (избирательность: событий нет).
